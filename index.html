• git config --global user.name blabla: Ця команда встановлює глобальне ім'я користувача для вашої конфігурації Git.
  • git config --global user.email blabal@bla: Ця команда встановлює глобальну електронну адресу користувача для вашої конфігурації Git.
  • git init: Ця команда ініціалізує новий репозиторій Git в поточній директорії, створюючи нову піддиректорію .git, в якій зберігатимуться дані системи контролю версій.
  • git status: Ця команда показує поточний стан вашого репозиторію Git, включаючи змінені, проіндексовані або невідстежувані файли.
  • git add file: Ця команда додає файл або зміни до наступного коміту. Ви можете вказати імена окремих файлів або використати git add ., щоб додати всі зміни в поточній директорії.
  • git commit: Ця команда створює новий коміт, зберігаючи зміни, які ви проіндексували за допомогою git add, в репозиторії Git. Вона відкриває текстовий редактор, в якому ви можете ввести повідомлення коміту, пояснюючи зміни.
  • git log: Ця команда відображає журнал всіх комітів у репозиторії, показуючи хеш коміту, автора, дату та повідомлення коміту для кожного з них.
  • git checkout branch: Ця команда перемикається на вказану гілку у вашому репозиторії Git. Вона дозволяє переміщуватись між гілками або створювати нові гілки.
  • git branch - виводить список усіх гілок у репозиторії або створює нову гілку.
  • git push - завантажує зміни з локального репозиторію на віддалений сервер.
  • git pull - отримує оновлення з віддаленого сервера і об'єднує їх з локальним репозиторієм.
  • git merge - об'єднує зміни з однієї гілки в іншу.
  • git clone - копіює віддалений репозиторій на локальний комп'ютер.
  • Файл .gitignore використовується в Git для вказівки файлів та папок, які необхідно ігнорувати при виконанні операцій коміту (git commit) та інших операцій Git.
  • git revert - створює новий коміт, який відміняє зміни, зроблені у попередньому коміті.
  • git reset - скидає стан репозиторію на певний коміт, видаляючи або переміщуючи коміти.
  • git rebase - перебазує (змінює базовий коміт) гілку на іншу гілку або коміт, щоб включити останні зміни.
  • git cherry-pick - вибирає окремий коміт з історії і застосовує його зміни до поточної гілки.
  • git stash - зберігає тимчасові зміни у стеші (збереженні) для повернення до них пізніше, щоб можна було переключитись на іншу гілку чи виконати інші операції.
